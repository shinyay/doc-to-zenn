## 1. 3 つの move

> **全最適化は 3 つの 1 つに変装している**

| # | Move | 典型例 |
|---|------|--------|
| 1 | **より少なく送る** | システムプロンプト trim、context scoping、履歴要約、未使用ツール drop |
| 2 | **より少なく受ける** | terse 指示、固定 JSON スキーマ、preamble なし、長さ上限 |
| 3 | **トークンあたり払う額を減らす** | 安いモデル段、prefix キャッシュ設計、合うワークフロー mode |

## 2. 7 技法ファミリ — 採用順

| 順 | ファミリ | 努力 | インパクト |
|----|----------|------|-----------|
| 1 | **Output control** | 極低 | 高 |
| 2 | **Context scoping** | 低〜中 | 高 |
| 3 | **Prompt compression** | 中 | 中 |
| 4 | **Model tiering** | 中 | 高 |
| 5 | **Caching 規律** | 低 | 高（安定 prefix なら） |
| 6 | **Tool / MCP 規律** | 低 | 中 |
| 7 | **Workflow 選択** | 継続 | 判断あたり高 |

**ルール**: パラメータダイアル（temperature 等）から始めない。**構造仕事を先に**。

## 3. 10 の hygiene 原則（第12章）

```
1.  あらゆるトークンを正当化させよ      "なぜここに？"と問えない行は削除候補
2.  最も安定したものを最初に           cache hit を最大化
3.  揮発内容は最後に                    安定 prefix を破壊しない
4.  Declarative > Imperative           "Output: JSON" が "必ず JSON で..." に勝つ
5.  繰り返しを 1 箇所に集約             同じ意図を 3 箇所書かない
6.  死んだルールを定期削除               起きないシナリオの指示は dead weight
7.  ツールはスコープしてロード           全ワークフローに全ツールを exposing しない
8.  履歴は要約 → 切る                  古い turn は復元不能でも切る価値
9.  測ってから触る                      推測ではなくメトリックで判断
10. 規律を継続                          1 度の clean-up は 3 ヶ月で drift する
```

## 4. 14 アンチパターン早見（第14章）

| カテゴリ | アンチパターン |
|----------|--------------|
| プロンプト | 礼儀的 padding / 同じ指示を 3 回 / dead ルール残置 |
| コンテキスト | 全ワークフロー指示を 1 ファイル / RAG 取り過ぎ |
| ツール | 未使用 MCP 接続放置 / 重複ツール定義 |
| モデル | 全部最高 tier / router なし |
| ワークフロー | one-shot 質問に agent loop / agent タスクを ask で |
| 計測 | 平均だけ見る / バケット別に見ない |
| キャッシュ | 揮発トークンを位置 1 / per-user 変数を先頭 |

## 5. 計測の 4 トークン数（第13章）

```
┌──────────────┬─────────────────────────────────┐
│ Input        │ プロンプト全体（隠れ消費者含む） │
│ Output       │ モデルが生成した分               │
│ Reasoning    │ 内部思考（vendor により請求対象）│
│ Cached       │ prefix cache hit 分（割引対象）  │
└──────────────┴─────────────────────────────────┘
```

**見るべき統計**: 平均 ❌ → **p50 / p95 / p99** ⭕

## 6. 四半期トークン audit checklist（第16章）

```
□ 1. 前四半期メトリックを pull
□ 2. Top 3 コスト drivers を特定
□ 3. システムプロンプトを声に出して再読
□ 4. ツールリスト — 90 日未使用を削除
□ 5. モデル段 vs タスク難度を再 check
□ 6. Cache-friendly prefix 構造を verify
□ 7. 次四半期の実験 1 つを記録
```
